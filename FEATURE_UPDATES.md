## Example Entry Format

When adding new entries, use the following format:

- New features, files, or capabilities

- Changes to existing functionality

### Deprecated
- Features that will be removed in future releases

### Removed
- Features or files that have been removed

### Fixed
- Bug fixes

### Security
- Security vulnerability patches

## How to Use This File

### For Contributors
When submitting a pull request, add your changes under the `[Unreleased]` section in the appropriate category (Added, Changed, Fixed, etc.). This helps maintainers track what's included in each release.

### For Maintainers
Before releasing a new version:
1. Review all entries under `[Unreleased]`
2. Create a new version section (e.g., `## [1.0.0] - 2025-11-22`)
3. Move entries from `[Unreleased]` to the new version section
4. Add a link to the version comparison at the bottom of the file
5. Update the version number in the project (if applicable)
